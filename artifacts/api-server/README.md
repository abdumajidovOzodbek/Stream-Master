# Telegram Channel Video Bridge

Logs into Telegram **as a user** (not a bot) via the MTProto API (gramjs), fetches videos from any channel you've joined, downloads them locally, and serves them as plain HTTP URLs through an Express API.

## Project Structure

```
src/
├── auth/        # CLI login flow — generates a StringSession
├── telegram/    # gramjs client + channel/video fetching + downloads
├── server/      # Express routes for /channel-videos and /videos/{file}
└── routes/      # Router wiring
storage/         # Downloaded video files (served at /api/videos/*)
```

## Required Secrets

| Secret               | How to get it |
| -------------------- | ------------------------------------------------------- |
| `TELEGRAM_API_ID`    | https://my.telegram.org/apps → create app → `api_id`    |
| `TELEGRAM_API_HASH`  | https://my.telegram.org/apps → create app → `api_hash`  |
| `TELEGRAM_SESSION`   | Generated by the login script below (see step 1)        |

## Setup

### 1. Generate a session string

Run the interactive login script. It will prompt for your phone number, the SMS/Telegram code, and an optional 2FA password.

```bash
pnpm --filter @workspace/api-server run login
```

When it succeeds, it prints a long `StringSession` — **copy it and save it as a secret named `TELEGRAM_SESSION`**. You only need to do this once.

### 2. Restart the server

After adding `TELEGRAM_SESSION`, restart the API Server workflow. The first request will connect and stay connected.

## Endpoints

All endpoints are mounted under `/api`.

### `GET /api/channel-videos?channel=<username|id>`

Fetches recent messages from the channel, filters videos, downloads any that aren't already cached, and returns JSON:

```json
{
  "channel": "telegram",
  "count": 2,
  "videos": [
    {
      "messageId": 12345,
      "url": "http://localhost/api/videos/12345.mp4",
      "fileName": "12345.mp4",
      "fileSize": 8421234,
      "duration": 47,
      "mimeType": "video/mp4",
      "date": 1756123456,
      "caption": "Demo clip",
      "downloaded": true
    }
  ]
}
```

Optional query params:

- `limit` — max messages to scan (default 20, max 100)
- `download=false` — only return metadata, do not download

### `GET /api/videos/{filename}`

Streams a previously downloaded video file from `storage/`. Filenames are `{messageId}.{ext}`.

## Behavior Notes

- Files already on disk are not re-downloaded.
- Errors are mapped to sensible HTTP status codes:
  - `400` — missing `channel` query param
  - `401` — `TELEGRAM_SESSION` missing
  - `403` — channel is private / username invalid / no access
  - `500` — anything else
- All async I/O uses `async/await`. Logging uses the shared pino logger (`req.log` in routes, `logger` elsewhere).
- Direct Telegram CDN URLs are intentionally NOT used — they are not publicly accessible. This service downloads files via MTProto and re-serves them.
